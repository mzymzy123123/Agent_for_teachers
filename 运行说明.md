## 三、启动前端服务
## node.js版本：node-v18.19.0-linux-x64
**在第二个终端窗口执行：**

```bash
cd /mnt/pfs_l2/jieti_team/LA/mazeying/competition/frontend

# 如果是第一次运行，需要先安装依赖
npm install

# 配置后端地址（重要！）
# 如果后端运行在本地：可以跳过这一步，使用默认值 http://localhost:8000
# 如果后端运行在远程服务器：需要创建 .env 文件并设置服务器地址
cat > .env << 'EOF'
VITE_API_BASE_URL=http://你的服务器IP:8000
EOF
# 例如：VITE_API_BASE_URL=http://192.168.1.100:8000
# 或者：VITE_API_BASE_URL=http://your-server.com:8000

# 启动前端开发服务器
npm run dev
```

看到以下输出说明前端启动成功：
```
  VITE v5.x  ready in xxx ms
  ➜  Local:   http://localhost:5174/
```

**保持这个终端窗口也不要关闭！**

**注意**：如果后端运行在远程服务器上，前端需要知道服务器的 IP 地址或域名。可以通过以下方式获取：
- 在服务器上运行：`hostname -I` 或 `ip addr show` 查看服务器 IP
- 或者使用服务器的域名（如果有）

## 五、使用流程

1. **上传视频**：在页面中选择一个授课视频文件（支持 mp4、mov 等常见格式）
2. **开始分析**：点击"开始分析"按钮
3. **等待结果**：系统会自动进行：
   - 音频提取与分析（音量、音调、停顿等）
   - 视频视觉分析（背板检测、动作频率等）
   - 语音转文字（ASR）
   - LLM 内容分析（教学内容结构、逻辑、互动等）
   - 综合评分生成
4. **查看报告**：页面会展示：
   - 总体评级（好/中/差）
   - 外功评分（表现力维度）
   - 内功评分（教学内容维度）
   - 详细 JSON 数据

## 六、常见问题排查

### 后端报错
- **500 Internal Server Error**：检查环境变量 `TAL_MLOPS_APP_ID` / `TAL_MLOPS_APP_KEY` 是否正确设置
- **ASR service error (server side)**：公司 ASR 服务暂时不可用，系统会自动使用占位文本继续流程
- **ModuleNotFoundError**：运行 `pip install -r requirements.txt` 安装缺失的 Python 包

### 前端报错
- **页面打不开**：确认前端终端里 `npm run dev` 正在运行，且端口 5173 未被占用
- **上传后报错**：打开浏览器开发者工具（F12）→ Network 面板，查看 `/api/evaluate` 请求的状态码和错误信息

### 前后端无法通信
- **如果后端运行在本地**：
  - 确认后端运行在 `http://localhost:8000`
  - 在浏览器访问 `http://localhost:8000/docs` 应该能看到 API 文档
- **如果后端运行在远程服务器**：
  - 确认后端运行在 `http://0.0.0.0:8000`（监听所有网络接口）
  - 获取服务器 IP 地址：在服务器上运行 `hostname -I` 或 `ip addr show`
  - 在 `frontend/` 目录下创建 `.env` 文件，设置 `VITE_API_BASE_URL=http://服务器IP:8000`
  - 重启前端服务（`npm run dev`）
  - 在浏览器访问 `http://服务器IP:8000/docs` 应该能看到 API 文档
  - 检查防火墙是否开放了 8000 端口

## 七、快速测试后端接口（不启动前端）

如果想直接用命令行测试后端是否正常：

```bash
curl -X POST "http://localhost:8000/api/evaluate" \
  -H "Accept: application/json" \
  -F "file=@/path/to/your/video.mp4" \
  | python3 -m json.tool
```

如果返回完整的 JSON 报告，说明后端运行正常。
